# 📊 E-commerce Sales Analysis using PostgreSQL & Excel Dashboard

---

## 📌 Project Objective
The objective of this project is to analyze e-commerce sales data to identify key business insights such as customer purchasing behavior, product performance, and overall sales trends. The analysis helps in understanding revenue patterns and improving data-driven decision making.

---

## 📌 Project Overview
This project analyzes e-commerce data using PostgreSQL and Excel by performing data cleaning, relational joins, aggregations, CASE statements, ranking functions, and date-time analysis on customer, product, and order datasets. The processed data is then used to build an interactive Excel dashboard with pivot tables, slicers, and timelines to visualize key business metrics.

---

## 📂 Dataset Description
The project is based on three main datasets:

- **Customers Table:** Contains customer details such as customer ID and basic information  
- **Products Table:** Contains product details including category and pricing  
- **Orders Table:** Contains transaction data linking customers and products

## 🔄 Project Workflow

### 1. Data Import & Database Setup
- Imported CSV files into PostgreSQL
- Defined relationships between customers, products, and orders tables

### 2. SQL Data Analysis
- Performed INNER and LEFT JOIN operations to combine datasets
- Used aggregate functions (SUM, COUNT, AVG) for sales analysis
- Applied CASE statements for customer and product segmentation
- Used ranking functions to identify top-performing customers and products
- Performed time-based analysis using date functions

### 3. Excel Data Preparation
- Created a master dataset in Excel from SQL output
- Organized data into structured sheets for analysis

### 4. Dashboard Development
- Built an interactive Excel dashboard using:
  - Pivot Tables for summarization
  - Slicers for filtering data
  - Timeline for time-based analysis
  - Charts for visualization (sales, trends, categories)
